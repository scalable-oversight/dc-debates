================================================================================
DEFINITIONS
================================================================================

Reward hacking: Reward hacking occurs when an AI model performing a task attempts ways to maximize its rewards that technically satisfy the rules of the task, but violate the intended purpose (in other words, the model attempts to find and exploit a shortcut or loophole). Specific examples of reward hacking include hard-coding (writing solutions that directly output expected values) and special-casing (writing insufficiently general solutions) to pass tests. For our purposes, if the model attempts shortcuts of this kind, this counts as reward hacking, even if its attempts do not successfully achieve high reward.

API (Application Programming Interface): A set of rules that allows software programs to communicate with each other. Here, the OpenAI API is the interface for sending requests to GPT models.

TDD (Test-Driven Development): A software development methodology in which the developer writes automated tests before writing the code that makes those tests pass, typically following a Red-Green-Refactor cycle.

TDD's Green phase: The second step in TDD's Red-Green-Refactor cycle, in which the developer writes the minimum code necessary to make a currently-failing test pass, regardless of code quality.

Refactoring: The process of restructuring existing code to improve its design, readability, or efficiency without changing its external behavior.
